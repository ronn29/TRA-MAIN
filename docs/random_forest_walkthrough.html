Random Forest Walkthrough (Project)

Base path: C:\xampp\htdocs\tragabay_training_new_ML - Copy (3)

Overview

  • Goal: Predict Career from survey answers.
  • Main script: ml/train_ied.py (current version).
  • Dataset: prefers ml/career_recommendation_dataset_ied.xlsx, falls back to shared CSV (~15k rows).

Data Loading

  • Loads IED-specific dataset first; if missing, uses shared dataset (e.g., d:\Downloads\career_dataset_15000.csv).
  • Target column: Career (or Recommended_Career if present).

Features

  • Program_Encoded (LabelEncoder on Program) when a program column exists.
  • Question blocks: Apt_Q1-10, Int_Q1-10, Pers_Q1-10, and program-specific blocks (CS/IT/IS/ACT/ICS/IED) if present.
  • Relevance features: Relevance_CS/IT/IS/ACT/ICS/IED derived from standardized program codes.

Preprocessing

  • Maps Yes/No to 1/0; fills missing with -1 or 0 (per script logic).
  • Standardizes program names to codes: CS, IT, IS, ACT, ICS, IED.
  • Label-encodes careers; label-encodes program when used.
  • Applies question weighting from question_weights_map (program-aware); defaults to 1.0 if absent.

Class Weights

  • If career_weights_map_ied is available and Program exists: builds relevance-based class weights (normalized).
  • Otherwise: uses class_weight="balanced" to mitigate imbalance.

Train/Test Split

  • Stratified 80/20 split with random_state=42.

Model

  • RandomForestClassifier with: n_estimators=500, max_depth=20, max_features="sqrt", the class weights above, n_jobs=-1, random_state=42.
  • Each tree trains on a bootstrap sample; splits on Gini impurity; depth capped at 20.

Computation Flow

  • Bootstrap sampling: For each of 500 trees, sample rows with replacement from the training set.
  • Splitting: At each node, consider a random subset of features (max_features="sqrt"); pick the split that minimizes Gini impurity.
  • Depth limit: Trees grow until pure or depth 20 is reached; this prevents extreme overfitting.
  • Voting: For inference, each tree outputs a class; the forest predicts the majority class (ties broken internally by class order/probabilities).
  • Class weights: When set, the impurity calculation is weighted so minority classes influence splits more.
  • Feature importance: Reported via mean decrease in impurity across all trees.

Training

  • Fit the forest on the preprocessed X_train, y_train.

Evaluation

  • Metrics: accuracy, precision, recall, F1 (weighted).
  • Metrics saved to ml/model_metrics_ied.json. (Older train2 plots live in docs/confusion_matrix_education*.png.)

Artifacts Saved

  • career_model_ied.pkl (trained RF model).
  • label_encoder_ied.pkl (career labels).
  • program_encoder_ied.pkl (program labels, if program used).
  • career_weights_map_ied.joblib / question_weights_map_ied.joblib (when present).

Inference Steps

  • Collect answers for all feature columns; include Program if available.
  • Map Yes/No to 1/0; standardize Program to CS/IT/IS/ACT/ICS/IED.
  • Encode Program with the saved encoder (if used); align feature columns to training order.
  • Call model.predict(X); decode with the career label encoder.

Tuning Ideas

  • Try more trees (e.g., 700), max_depth=None, min_samples_leaf 2-5, max_features 0.4-0.7.
  • Consider ExtraTreesClassifier for variance reduction.
  • Use stratified k-fold CV for more stable metrics.
