Random Forest Training & Prediction (Project)

Base path: C:\xampp\htdocs\tragabay_training_new_ML - Copy (3)

Key Files

  • Training script: ml/train_ied.py (uses RandomForestClassifier).
  • Prediction script (CLI): ml/predict_career_ied.py.
  • Web prediction: admin/includes/model_predict.php (calls the saved model).
  • Artifacts after training: ml/career_model_ied.pkl, label_encoder_ied.pkl, program_encoder_ied.pkl, question/class weight files.

What the Model Learns

  • Goal: predict a career label from survey answers and the student program.
  • Dataset preference: ml/career_recommendation_dataset_ied.xlsx; falls back to shared CSV (~15k rows) if missing.
  • Features: Program (encoded), aptitude/intention/personality blocks, program-specific question blocks (CS/IT/IS/ACT/ICS/IED when available), and relevance flags per program.
  • Target: Career (or Recommended_Career if present).

How Training Works

  • Load and clean data: map Yes/No to 1/0, fill gaps, standardize program codes (CS, IT, IS, ACT, ICS, IED).
  • Encode labels: LabelEncoder for careers; LabelEncoder for program when present.
  • Apply weights: question weights per program; class weights from career_weights_map_ied when available, else class_weight="balanced".
  • Split: stratified 80/20 train-test with random_state=42.
  • Train: RandomForestClassifier (n_estimators=500, max_depth=20, max_features="sqrt", bootstrap on, n_jobs=-1).
  • Evaluate: accuracy/precision/recall/F1 (weighted); metrics stored in ml/model_metrics_ied.json.
  • Save artifacts: model + encoders + weight maps to ml/; these are reused by PHP and the CLI predictor.

Run Training (on this system)

From the project root:

python ml/train_ied.py
  • Make sure Python environment in env/ is active or use system Python with dependencies installed (scikit-learn, pandas, numpy, joblib).
  • Outputs: model/encoders in ml/, metrics JSON for quick quality checks.

How Prediction Works

  • Load saved model and encoders (prefers IED artifacts, falls back to base ones).
  • Standardize the student program, encode it, and set relevance flags (one-hot for CS/IT/IS/ACT/ICS/IED).
  • Fill every expected feature column; apply question weights for the detected program when available.
  • Forest inference: each tree votes; majority vote (with probabilities) decides the career.
  • Decode the predicted index back to the career label; also produce top-5 probabilities.

Run a Prediction (CLI example)

python ml/predict_career_ied.py ^
  --input-json "{ \"current_program\": \"ICS\", \"features\": {
    \"Apt_Q1\": 1, \"Apt_Q2\": 0, \"Int_Q1\": 1, \"Pers_Q1\": 1
    /* ...include all question fields used in training... */
  }}"
  • Use the same feature names the trainer expects (see model.feature_names_in_ if unsure).
  • Web flow: PHP collects answers, builds the feature payload, calls the saved model, then displays the decoded career.

Quick Mental Model

  • Training: build many shallow-ish decision trees on bootstrap samples; each tree sees random feature subsets at splits to reduce correlation.
  • Prediction: send the aligned feature row through all trees; take the majority vote (probabilities = normalized votes).
  • Why it helps: bagging + feature randomness lowers variance and handles mixed binary/ordinal inputs without heavy scaling.
