Technical Documentation & Workflow Guide

Machine Learning-Based Career Prediction System

1. System Overview

The Career Recommendation System is an intelligent platform that uses Random Forest Machine Learning to provide personalized career recommendations to students based on their assessment responses. The system analyzes student answers from multiple assessments and generates career predictions tailored to their academic program.

Key Feature: The system requires students to complete exactly 40 assessment questions before generating recommendations, ensuring comprehensive data collection for accurate predictions.

2. System Architecture

The system consists of three main components:

Component Technology Purpose
Frontend PHP, HTML, JavaScript User interface, data collection, result display
Backend Processing PHP Data validation, feature extraction, result filtering
ML Engine Python (scikit-learn) Random Forest model inference, probability calculation

3. Complete Workflow

Student Authentication
Student logs in and accesses the career prediction page. System verifies student role and session.
Assessment Completion Check
System validates that student has completed all required assessments (exactly 40 answers). If incomplete, student is prompted to finish assessments first.
Data Collection
System retrieves all completed assessment answers from assessment_results_tbl where assessment_status = 'Completed'. Also fetches student's program information.
Feature Vector Construction
Answers are converted into a 40-feature vector:
  • Aptitude Questions: Apt_Q1 through Apt_Q10 (10 features)
  • Interest Questions: Int_Q1 through Int_Q10 (10 features)
  • Personality Questions: Pers_Q1 through Pers_Q10 (10 features)
  • Program-Specific Questions: CS_Q1-10, IT_Q1-10, IS_Q1-10, ACT_Q1-10 (10 features)
Each answer is normalized to binary: 1 for "Yes", 0 for "No" or missing.
Program Context Addition
System adds program-specific relevance features:
  • Program encoding (CS, IT, IS, or ACT)
  • Relevance scores for each program type
  • Question weights based on program-specific importance
ML Model Invocation
PHP creates a JSON payload and executes Python script:
{ "current_program": "BSCS", "features": { "Apt_Q1": 1, "Apt_Q2": 0, ... "ACT_Q10": 1 } }
Python script loads the trained Random Forest model and generates predictions.
Probability Calculation
Random Forest model calculates probability scores for all possible careers. Returns top 5 careers with their probability scores.
Career Weights Application
System applies career-program relevance weights to adjust probabilities:
  • Careers more relevant to student's program get higher adjusted scores
  • Weights are defined in career_weights_map.json
  • Each career has relevance scores (0-3) for CS, IT, IS, and ACT programs
Program Filtering
Results are filtered to match student's program:
  • Only careers allowed for student's program are kept
  • Missing allowed careers are appended to ensure program relevance
  • Duplicates are removed while preserving order
Result Generation
System generates final recommendation with:
  • Most Likely Career: Highest probability match
  • Least Likely Career: Lowest probability from filtered list (if 2+ careers)
  • Full Career List: Ordered list of all recommended careers
  • Career Descriptions: Detailed information for each career
Logging & Display
Prediction is logged to ml/prediction_results.jsonl for analytics. Results are displayed to the student with career descriptions and resources.

4. Machine Learning Model Details

4.1 Model Type: Random Forest Classifier

The system uses a Random Forest ensemble learning algorithm, which combines multiple decision trees to make predictions. This approach provides:

  • High Accuracy: Multiple trees reduce overfitting
  • Feature Importance: Identifies which questions are most predictive
  • Probability Scores: Provides confidence levels for each prediction
  • Robustness: Handles missing or noisy data well

4.2 Model Configuration

Parameter Value Description
n_estimators 500 Number of decision trees in the forest
max_depth 20 Maximum depth of each tree
min_samples_split 2 Minimum samples required to split a node
min_samples_leaf 1 Minimum samples required in a leaf node
max_features 'sqrt' Number of features to consider for best split
class_weight Balanced/Custom Weights classes based on career-program relevance

4.3 Training Data

The model is trained on a dataset containing student assessment responses and their corresponding career recommendations. The dataset includes:

  • Multiple student profiles with complete assessment data
  • Career labels for each student
  • Program information (CS, IT, IS, ACT)
  • Question weights for program-specific importance

5. Feature Engineering

5.1 Question Weights

The system applies question weights to emphasize the importance of certain questions based on the student's program:

  • Program-Specific Weights: Some questions are more important for specific programs
  • Universal Weights: Some questions have equal importance across all programs
  • Weights are defined in question_weights_map.py

5.2 Career-Program Relevance

Each career has a relevance score (0-3) for each program type:

  • 0: Not relevant to the program
  • 1: Somewhat relevant
  • 2: Moderately relevant
  • 3: Highly relevant

These scores are used to adjust prediction probabilities, ensuring careers more aligned with the student's program receive higher rankings.

6. Data Flow Diagram

Input → Processing → Output Flow:

Input: Student Assessment Answers (40 questions)
↓

↓
ML Processing: Random Forest prediction with probability scores
↓
Weight Application: Adjust probabilities based on career-program relevance
↓

↓

7. Error Handling

The system handles various error scenarios gracefully:

Error Type Condition Response
Incomplete Assessment Less than 40 answers Prompt student to complete all assessments
Python Runtime Missing Python executable not found Display error message, log issue
ML Runtime Failed Python script execution error Return error with stderr details
Invalid Model Output Malformed JSON response Return error, log for debugging
No Allowed Careers No careers match student's program Display message to contact administrator

10. Performance & Accuracy

Model Performance Metrics:
The Random Forest model is trained and evaluated using standard machine learning metrics:
  • Accuracy: Percentage of correct predictions
  • Precision: Accuracy of positive predictions
  • Recall: Ability to find all relevant careers
  • F1 Score: Harmonic mean of precision and recall
Model metrics are stored in ml/model_metrics.json and can be displayed to administrators.
